# Kokkuvõte

Git on versioonihaldussüsteem. Seda kasutatakse failide muudatuste jälgimiseks ja projektide haldamiseks. Git võimaldab näha, kes ja millal muudatusi tegi. Commit salvestab muudatused kohalikku ajalukku. Push saadab need muudatused GitHubi serverisse. Branch ehk haru võimaldab töötada uute muudatustega põhiversiooni mõjutamata. Harud muudavad meeskonnatöö turvalisemaks. Pull Request on ettepanek muudatuste ühendamiseks põhiharuga. See võimaldab muudatusi üle vaadata enne ühendamist. Git on mugavam kui failide kopeerimine USB-mälule, sest kogu ajalugu säilib. Git võimaldab vajadusel taastada varasema versiooni. GitHubi Issues aitavad hallata ülesandeid. Projektijuhtimises saab kasutada Issues funktsiooni tööde planeerimiseks ja jälgimiseks. GitHub toetab ka meeskonnatööd ning koostööd erinevate arendajate vahel. Selle töö käigus õppisin looma repositooriume, kasutama committe ja harusid. Kõige raskem osa oli Pull Requesti loomine. Kõige lihtsam oli README failide loomine. Tulevikus kasutan GitHubi kooliprojektide ja isiklike projektide hoidmiseks.
