# Teema 1: ITIL

## Põhimõisted

* ITIL – IT teenuste haldamise raamistik.
* Incident – teenuse katkestus või rike.
* Problem – intsidendi algpõhjus.
* Service Desk – kasutajatoe keskus.
* SLA – teenustaseme kokkulepe.

## Mõistete tabel

| Mõiste       | Selgitus                     |
| ------------ | ---------------------------- |
| Incident     | Teenuse häire                |
| Problem      | Häire põhjuse leidmine       |
| SLA          | Teenustaseme kokkulepe       |
| Service Desk | Kasutajatugi                 |
| ITIL         | Teenuste haldamise raamistik |

## Mida õppisin

Õppisin, mis on ITIL ja miks seda kasutatakse. Sain teada, kuidas IT teenuseid hallatakse. Mõistsin erinevust intsidendi ja probleemi vahel. Tutvusin Service Desk rolliga. Sain aru, miks SLA on oluline.

## Näide

Kui ettevõttes ei tööta e-post, registreeritakse incident ning kasutajatugi hakkab probleemi lahendama.
